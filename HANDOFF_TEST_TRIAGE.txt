🛡️ GUARDRAILS (Lessons from This Session)
==========================================

**Before Deleting ANY Test File:**
1. Check git: `git log --all -- "**/filename"`
2. Check references: `grep -r "key_pattern" ml/`
3. Check if it's WIP: Look for related env vars, feature flags
4. Default to skip, not delete

**Before Fixing ANY Test:**
1. Read the full error first
2. Check if it's testing real functionality or WIP
3. Verify the fix makes sense (don't just make test pass)

**Commit Strategy:**
- Commit working fixes immediately (don't batch)
- Use descriptive messages: "fix: X by doing Y"
- NO --no-verify unless pre-commit has branch-wide issues
