## 📊 Architecture Summary

### The Problem We're Solving

Current implementation has earnings data in a **standalone silo** (`EarningsStore`) that:
- ❌ Bypasses DataStore facade
- ❌ Has no contract validation
- ❌ Has no lineage tracking
- ❌ Has no progressive fallback
- ❌ Not accessible to actors

### The Solution (2-Layer Integration)

**Layer 1: Raw Earnings Data → DataStore + DataRegistry**
```
EDGAR/Yahoo → DataStore → EarningsStore (internal) → PostgreSQL
                  ↓
           DataRegistry (contracts, lineage, validation)
```

**Layer 2: Computed Features → FeatureStore + FeatureRegistry**
```
Raw Earnings → TransformSpec → FeatureStore → PostgreSQL
                                     ↓
                            FeatureRegistry (schema validation)
```

### Key Architectural Decisions

1. **Raw data through DataStore** - Earnings actuals/estimates are raw inputs
2. **Computed features through FeatureStore** - Surprise/growth/momentum are derived
3. **EarningsStore becomes internal** - Not exposed in public API
4. **Point-in-time at data layer** - Filtering with `as_of_ts` in DataStore
5. **Progressive fallback** - PostgreSQL → File → Dummy

## 📚 Key Patterns to Follow

### Pattern 1: Progressive Fallback
```python
# Always have fallback chain
try:
    store = EarningsStore(connection_string)  # PostgreSQL
except:
    try:
        store = FileEarningsStore(base_path)  # File
    except:
        store = DummyEarningsStore()  # In-memory
```

### Pattern 2: Protocol-First Design
```python
# Define protocol
class EarningsStoreProtocol(Protocol):
    def get_actuals(...) -> list[dict]: ...

# Implementations conform without inheritance
class FileEarningsStore:  # No inheritance
    def get_actuals(...) -> list[dict]: ...  # Conforms to protocol
```

### Pattern 3: Point-in-Time Correctness
```python
# Always filter by ts_event in data layer
actuals = data_store.get_earnings_actuals_at_or_before(
    ticker="AAPL",
    ts_event=backtest_ts,  # Only returns filings with ts_event < backtest_ts
    limit=5,
)
```

### Pattern 4: Contract Validation
```python
# Validate before write
data_store.write_earnings_actual(
    ticker="AAPL",
    eps_diluted=2.52,
    ts_event=filing_ts,
    # ... DataStore validates against EARNINGS_ACTUALS_CONTRACT
)
```

### Pattern 5: Event-Driven Observability
```python
# Every write emits event
data_store.write_earnings_actual(...)
# → Validates contract
# → Writes to store
# → Emits DataEvent
# → Updates watermark
# → Increments metrics
```
