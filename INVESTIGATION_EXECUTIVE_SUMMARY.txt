FINDING: The test contains an INVALID MATHEMATICAL PROPERTY

The assertion at line 639 claims:
  "Reweighting high-confidence signals (→weight 10) and low-confidence 
   signals (→weight 1) will always increase (or maintain) overall confidence"

This property is MATHEMATICALLY FALSE and can be violated by valid signal
distributions generated by hypothesis.

WHY IT'S INVALID:
  The property assumes that applying absolute weights (high→10, low→1) 
  will always improve confidence. However, this depends entirely on the 
  ORIGINAL weight distribution.
  
  If the original distribution already heavily weights high-confidence 
  signals, the reweighting operation can have NO EFFECT or can actually 
  REDUCE overall confidence.

EXAMPLE VIOLATION:
  Original signals:
    Signal 0: confidence=0.99, weight=100.0 (high-conf, super heavy)
    Signal 1: confidence=0.1,  weight=1.0   (low-conf, light)
  
  Original conf = (0.99×100 + 0.1×1) / (100+1) = 99.1/101 ≈ 0.981
  
  After reweighting (absolute: high→10, low→1):
    Signal 0: confidence=0.99, weight=10.0 (REDUCED from 100!)
    Signal 1: confidence=0.1,  weight=1.0  (unchanged)
  
  New conf = (0.99×10 + 0.1×1) / (10+1) = 9.99/11 ≈ 0.909
  
  VIOLATION: 0.909 < 0.981 (reweighting made things WORSE!)
